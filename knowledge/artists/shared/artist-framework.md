# Music Factory Artist Framework

## Purpose

This framework defines the standard structure for every Music Factory artist.

Future artists must follow this structure before they are rendered into website pages, prompts, album systems, archive systems, or visual systems.

## Standard Artist Structure

Every artist must define:

- Identity.
- Worldview.
- Visual System.
- Sound System.
- Keywords.
- Representative Quotes.
- Character Archive.
- Brand Positioning.
- Future Expansion Direction.

## Identity

Identity defines who the artist is inside Music Factory.

Identity includes:

- Artist name.
- Role.
- Realm.
- Core presence.
- Human or AI relationship.
- Guardian or performer status.

## Worldview

Worldview defines the world behind the artist.

Worldview includes:

- Realm logic.
- Emotional field.
- Cultural references.
- What the artist protects.
- What the artist refuses.

## Visual System

Visual System defines how the artist appears.

Visual System includes:

- Color direction.
- Photography direction.
- Image treatment.
- Costume or material logic.
- Typography mood.
- Motion behavior.

## Sound System

Sound System defines how the artist sounds.

Sound System includes:

- Vocal identity.
- Genre field.
- Instrument language.
- Production texture.
- Album logic.
- Forbidden sound references.

## Keywords

Keywords are the compact memory system for each artist.

They should support search, prompts, copywriting, and creative direction.

## Representative Quotes

Quotes should express the artist's worldview.

Quotes must feel authored, not generated.

## Character Archive

Character Archive records artist-related works, versions, albums, years, and development notes.

It is not a random list.

## Brand Positioning

Brand Positioning explains how the artist contributes to Music Factory as a cultural system.

## Future Expansion Direction

Future Expansion Direction defines where the artist can grow:

- Albums.
- MV.
- Performance.
- Website pages.
- Prompt systems.
- Archive systems.
- Cross-realm relationships.

## Agent Rule

When building or editing any artist, read this framework first, then read the artist-specific files.
