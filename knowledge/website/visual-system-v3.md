# Visual System V3

Music Factory Visual System V3.

## Goal

Upgrade Music Factory from:

Functional System

to:

Cinematic Civilization Experience.

## Global Rules

Visual System V3 unifies:

- Typography.
- Spacing.
- Color system.
- Layout.
- Motion.
- Navigation mood.

The system continues to use the existing architecture:

- Homepage V3.
- Artists Universe.
- Sound Civilization Map V2.
- Albums and Works.
- Archive Cinematic System.

No content data model change is required.

## Visual Principles

| Principle | Rule |
| --- | --- |
| Cinematic Mood | Pages should feel like entering rooms in one civilization. |
| Editorial Layout | Composition should be spacious, quiet, and intentional. |
| Slow UI Interaction | Movement should reveal attention, not excitement. |
| Human-scale Design | Navigation and reading must remain calm and usable. |
| Large Whitespace | Whitespace is part of the narrative system. |
| No Dashboard UI | Avoid product, SaaS, ecommerce, and data-interface language. |

## Page Direction

### Homepage

Homepage becomes a cinematic entrance:

- Large title.
- Breathing motion.
- Three civilization nodes.
- No product-style module language.

### Artists

Artists become portrait-led personality worlds:

- Portrait dominates the first impression.
- Text becomes quieter.
- Works are presented as story entries.
- The page must not feel like a catalogue list.

### Sound Map

Sound Map becomes a civilization landscape:

- Region spaces.
- Entering civilization zones.
- Spatial story reveal.
- No graph or topology language.

### Works and Albums

Works and Albums become editorial magazine chapters:

- Fewer visual boxes.
- Larger title rhythm.
- Large image / cover presence where available.
- Clear routes back to Artist, Map, and Archive.

### Archive

Archive becomes a museum memory space:

- Timeline feels spatial.
- Records feel like memory traces.
- It must not read as a data list.

## Implementation Rule

The V3 redesign is implemented as a shared visual layer:

`assets/visual-system-v3.css`

This keeps system architecture and data flow stable while aligning page-level visual language.
