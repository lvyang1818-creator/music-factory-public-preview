# Music Factory Knowledge Base V1

## Purpose

This knowledge base is the first unified project knowledge system for Music Factory.

All future AI Agents must read this knowledge base before making decisions about brand, artists, website, vision, copywriting, prompts, technology, or archive work.

## Operating Rule

Knowledge comes before implementation.

Before changing website pages, visual systems, content, prompts, data structures, or development behavior, an AI Agent must first review the relevant file in `knowledge/`.

## Structure

| Directory | Purpose |
| --- | --- |
| `brand/` | Mission, positioning, and philosophy. |
| `artists/` | Artist knowledge system, shared artist framework, and individual artist files. |
| `website/` | Information architecture, user journey, Sound Map, relationship model, navigation graph, world system, and product experience system. |
| `vision/` | Long-term cultural and artist-system vision. |
| `copywriting/` | Tone of voice and headline rules. |
| `prompt/` | Prompt systems for image, UI, and artist generation. |
| `technology/` | Project structure and frontend architecture. |
| `archive/` | Archive definition and memory rules. |

## Artist Knowledge Reading Order

2. `knowledge/artists/shared/artist-template.md`
3. Target artist `profile.md`
4. Target artist `worldview.md`
5. Target artist `visual-language.md`
6. Target artist `music-language.md`
7. Target artist `quotes.md`
8. Target artist `archive.md`

AI Agents must not invent official artist facts when the artist knowledge file is empty, incomplete, or marked as draft. Ask for approval or write a clearly marked draft only when authorized.

## Permanent Rule

Music Factory is not managed as loose webpage copy. It is managed as a cultural knowledge system, and the website is only one rendering surface.
