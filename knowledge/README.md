# Music Factory Knowledge Base

## Current Direction

Music Factory is being rebuilt as a simple Chinese music browsing website.

The previous complex Civilization / Cinematic V3 system is retired and must not guide future website work.

## Active Product Goal

The website should help visitors quickly do four things:

1. Understand what Music Factory is.
2. Enter one of the three AI singer pages.
3. Listen to songs.
4. Browse works, albums, and archive records when needed.

## Active Website Structure

Use this simple route model:

Homepage -> Singer -> Songs -> Song Detail / Album -> Archive

The homepage may keep the phrase "声音地图" only as a simple browsing idea, not as a complex system.

## Required Website Rules

- Chinese first.
- Simple navigation.
- Short copy.
- Clear singer introductions.
- Direct song display.
- Visible audio player.
- Fewer clicks.
- Mobile readable.
- No hidden core content.

## Retired Website Rules

Do not use these as active guidance:

- Cinematic Civilization Experience.
- Complex Sound Civilization Map architecture.
- Living Atlas / Memory Space as the main user path.
- Multi-layer civilization route logic.
- English-first copywriting.
- Heavy motion.
- Abstract museum navigation that hides songs.

## Data Rules

Keep useful data sources:

- `songs-data.js` for public song and audio data.
- Existing artist HTML pages as references for singer bios and song lists.
- Existing covers and audio links.

Do not invent new official songs or singer facts without user approval.

## Agent Reading Order

For future website rebuild work, read:

1. `PROJECT_MANUAL.md`
2. `knowledge/README.md`
3. `songs-data.js`
4. Existing artist pages under `artists/`
5. Existing assets only as needed

Do not start from the retired V3 civilization documents unless the user explicitly asks to inspect old history.

## Permanent Rule

Practical usefulness is more important than conceptual complexity.
