# Codx Development Rules

版本：V1.0

---

# 核心原则

优先复用。

禁止重复开发。

组件优先。

文档优先。

可维护优先。

性能优先。

---

# 开发流程

阅读 docs

↓

阅读 Knowledge Base

↓

阅读 Architecture

↓

开始开发

↓

更新文档

↓

提交 Commit

---

# 每次开发必须遵守

不得破坏现有页面。

不得修改无关模块。

不得影响已有组件。

不得写死数据。

所有数据来自 CMS。

---

# 组件开发

优先查找已有组件。

没有才允许新增。

新增必须可复用。

---

# 页面开发

页面仅负责布局。

业务逻辑进入 Feature。

UI 放入 Components。

---

# 修改原则

最小修改原则。

保持兼容。

不得推翻已有架构。

---

# 文档同步

新增组件：

更新组件文档。

新增页面：

更新页面文档。

新增接口：

更新 API 文档。

新增数据库：

更新数据库文档。

---

# Commit

一个功能一次 Commit。

禁止一个 Commit 修改多个独立需求。

---

# 长期目标

保持项目十年以上持续维护能力。
