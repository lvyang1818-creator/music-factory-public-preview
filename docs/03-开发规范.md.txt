# Development Specification

> Music Factory 开发规范

---

# 开发理念

所有开发以长期维护为第一原则。

代码不仅要能运行，更要易读、易改、易扩展。

---

# 技术栈

Framework

React

Language

TypeScript

Style

Tailwind CSS

Animation

Framer Motion

Package Manager

pnpm

---

# 文件命名

统一使用 PascalCase。

正确：

Hero.tsx

ArtistCard.tsx

AlbumCover.tsx

MusicPlayer.tsx

错误：

hero.tsx

hero_new.tsx

card1.tsx

---

# 文件夹命名

统一小写。

例如：

components

pages

assets

hooks

utils

services

---

# 一个组件一个文件

禁止一个文件放多个组件。

---

# 单个文件长度

建议：

300 行以内

最大：

500 行

超过立即拆分。

---

# CSS规范

禁止内联样式。

统一使用 Tailwind。

禁止重复 CSS。

---

# 颜色规范

禁止直接写颜色。

统一引用设计规范。

错误：

text-red-500

正确：

使用项目统一颜色变量。

---

# 间距规范

统一使用设计系统。

禁止随意：

margin-top:17px

padding-left:29px

所有间距遵循 UI 设计规范。

---

# 图片规范

全部放入 assets。

禁止散落。

按模块分类。

---

# SVG规范

统一放：

assets/icons

---

# Import规范

第三方库

↓

项目组件

↓

hooks

↓

utils

↓

style

保持统一顺序。

---

# Git规范

每完成一个功能：

提交一次。

Commit 信息简洁明确。

例如：

feat: Hero Section

fix: Music Player

style: Footer Layout

---

# 禁止事项

禁止复制粘贴代码。

禁止魔法数字。

禁止重复组件。

禁止硬编码。

禁止超过三级嵌套。

禁止无意义变量名。

---

# 开发目标

代码应达到：

可维护

可扩展

可复用

可阅读

长期稳定。
