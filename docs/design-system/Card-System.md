# Card System

## Purpose

Cards are used to frame repeated cultural objects: artists, albums, works, archive entries, and realm entrances. Cards must not become generic SaaS feature blocks.

## Card Principles

- A card represents an object, not decoration.
- Cards must carry content hierarchy.
- Cards should not be nested inside other cards.
- Cards should not be used for every section.

## Card Types

| Type | Role |
| --- | --- |
| Artist Card | Presents an artist as a world entrance. |
| Album Card | Presents an album as an artifact. |
| Work Card | Presents a work with cover, title, and minimal metadata. |
| Realm Card | Presents a realm as a museum navigation object. |
| Archive Card | Presents structured memory without spreadsheet feeling. |

## Visual Rules

- Use restrained borders or none.
- Use shadow only when it suggests object depth.
- Avoid glass panels.
- Avoid colorful feature-card stacks.
- Keep radius modest.

## Hover Rules

- Hover may raise by a few pixels.
- Hover may reveal a small action.
- Hover may increase cover clarity.
- Hover must not bounce, glow, rotate, or scale aggressively.

## Content Rules

- Title first.
- Metadata second.
- Description only when it adds meaning.
- Avoid overfilling cards with labels.

## Forbidden

- SaaS feature cards.
- Dashboard tiles.
- Nested card stacks.
- Decorative cards with no content role.
- Heavy shadows used to fake premium quality.

## Review Checklist

- Is the card necessary?
- Does it represent a meaningful object?
- Is it quiet and readable?
- Does it avoid product-template behavior?
