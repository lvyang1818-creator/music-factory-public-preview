# Image System

## Purpose

Images in Music Factory must reveal artists, albums, worlds, archives, and cultural objects. Images are never generic decoration.

## Image Principles

- Use images to provide identity, context, material, or memory.
- Avoid stock-like atmosphere.
- Avoid abstract AI imagery unless it is approved as part of a realm.
- Image treatment must feel editorial and human.

## Image Types

| Type | Role |
| --- | --- |
| Hero Background | Only when the spec approves an immersive visual. |
| Artist Image | Shows the human or guardian presence. |
| Album Cover | Shows the work as an artifact. |
| Realm Image | Communicates world atmosphere. |
| Archive Image | Supports memory and catalog identity. |

## Cropping Rules

- Preserve subject visibility.
- Avoid accidental face or text cropping.
- Use aspect ratio intentionally.
- Do not stretch images.
- Do not blur images to hide weak composition.

## Cover Rules

- Album covers should feel like cultural objects.
- Covers need enough size to be inspected.
- Covers should not be treated as tiny app thumbnails by default.
- Cover hover may reveal a play/action affordance quietly.

## Texture Rules

- Paper texture may be used subtly.
- Noise must be very weak.
- Texture must support material quality.
- Do not use visible grid lines as fake design.

## Forbidden

- Stock placeholders as final visuals.
- Abstract tech spheres.
- AI-glow backgrounds.
- Dark blurred hero images when inspection matters.
- Decorative image blocks without meaning.

## Review Checklist

- Does the image reveal something real or meaningful?
- Does the crop respect the subject?
- Does the image support the editorial rhythm?
- Does it avoid AI-template feeling?
