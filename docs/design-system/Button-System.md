# Button System

## Purpose

Buttons in Music Factory must be quiet, precise, and rare. The brand should not feel like a conversion funnel.

## Button Philosophy

- Use buttons only when an action is truly primary.
- Prefer text links for editorial navigation.
- Avoid oversized CTA blocks.
- Avoid gradient, glow, bounce, and sales-page button treatments.

## Button Types

| Type | Use |
| --- | --- |
| Text Link | Editorial exploration, section navigation, quiet calls to action. |
| Minimal Button | Interface action requiring clear affordance. |
| Archive Action | Filtering, opening, or inspecting archive records. |
| External Link | Leaving the site or opening a platform. |

## Visual Rules

- Background should usually be transparent, paper, or charcoal.
- Border radius should be controlled and not overly soft.
- Hover should use underline, opacity, or subtle border change.
- Scale hover is discouraged.
- Glow hover is forbidden.

## CTA Rules

- CTA copy must be short.
- CTA should not dominate the page unless the design spec explicitly demands it.
- CTA placement should follow reading rhythm.
- CTA should not make Music Factory feel like SaaS onboarding.

## Accessibility Rules

- Text must remain readable.
- Focus states must be visible.
- Touch targets must be usable on mobile.
- Buttons and links must have distinct roles.

## Forbidden

- Gradient buttons.
- Glowing buttons.
- Massive rounded SaaS CTAs.
- Bouncy hover.
- Sales funnel button stacks.

## Review Checklist

- Does the button feel editorial?
- Is the action necessary?
- Is hover quiet?
- Does the CTA avoid startup style?
