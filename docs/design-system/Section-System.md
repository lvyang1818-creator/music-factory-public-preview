# Section System

## Purpose

Sections create the rhythm of the Music Factory website. Each section must behave like a room in a cultural house, not a block in a landing page template.

## Section Principles

- Each section needs a clear role.
- Each section needs its own rhythm.
- Sections should not all use the same card layout.
- Sections must preserve editorial pacing.

## Section Types

| Type | Role |
| --- | --- |
| Hero | Establishes atmosphere and brand identity. |
| Philosophy | Defines cultural position. |
| Realms | Opens the artist universe. |
| Latest Releases | Presents current work as editorial selection. |
| Archive | Shows memory and continuity. |
| Footer | Closes the brand experience with orientation. |

## Layout Rules

- Use full-width fields rather than floating page cards.
- Use constrained inner containers for readability.
- Allow whitespace to carry emotional weight.
- Use asymmetry when it supports hierarchy.

## Section Transitions

- Transitions between sections should feel smooth and inevitable.
- Avoid abrupt color changes unless meaningful.
- Avoid decorative separators.
- Use spacing, typography, and subtle background changes.

## Forbidden

- Landing page feature sections.
- Repetitive card walls.
- In-app dashboard panels.
- Sections that exist only to fill scroll length.

## Review Checklist

- Does the section have a clear purpose?
- Does it feel like Music Factory?
- Does it preserve museum rhythm?
- Is it visually distinct without over-decoration?
