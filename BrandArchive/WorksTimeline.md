# Works Timeline

## Purpose

This file records the Milestone 16 Archive update for Works.

All works in the content-layer Works System enter Archive as timeline records.

## Timeline Rule

Each timeline record must preserve:

- Work title.
- Artist.
- Realm.
- Album.
- Status or date.
- Knowledge source.
- Return route to Sound Civilization Map.

## Milestone 16.5 Archive Definition

Archive is a content system made of:

- Memory Records.
- Works Timeline.
- Civilization Entries.

Archive must never be treated as a simple file list.

Every Work must preserve:

- Title.
- Artist Source.
- Time / Era.
- Narrative Description.
- Archive Link.

## Milestone 20 Archive Cinematic Rule

Archive is Music Civilization Memory Space.

The Archive page must render Works, Albums, Artists, and Sound Map relationships as a living memory system, not as a file list or table.

Each archive record must preserve:

- Source Artist.
- Source Work.
- Time or era.
- Civilization-context description.
- Related Sound Map node.
- Route to Artist Universe.
- Route to Work.
- Route to Sound Civilization Map.
- Route back to Homepage.

The legal data source remains:

Artists Knowledge Base -> Works Catalog -> Sound Map -> Archive.

Archive UI must consume catalog-derived records. It must not invent separate UI-only records.
